概要

mdocはMarkdown風の表記を持つ軽量マークアップ言語です。

Markdownは大変便利な表記ですが、 空白行あり時の動作など方言も多種存在し、 綺麗に表示させるため結局はHTMLの確認が必要になり、 既に軽量マークアップ言語を導入している場合、 ドキュメント形式をMarkdown法とするメリットを感じません。

mdocではミスしやすい構文をできるだけ排除し、 変換結果のテスト時間を低減します。

Markdownとは下記の差異があります。

  • 見出し1、見出し2の代替記法の廃止
  • 水平線は-のみで3文字以上以外の記法を廃止
  • アンダースコアによる強調機能の廃止
  • HTML記法の廃止
  • 段落以外で空行を要求されないよう変更
  • 定義リスト(HTMLのdl/dt/ddタグ)をサポート

リスト

 
* 項目1
	1. 項目1-1
	2. 項目1-2
+ 項目2
	* 項目2-1
	* 項目2-2
- 項目3
  • 項目1
    1. 項目1-1
    2. 項目1-2
  • 項目2
    • 項目2-1
    • 項目2-2
  • 項目3

mdocではリストのヘッド文字である*/+/-は同じ意味ですが、 HTMLへ変換後も表示はCCSのクラスで区別します。

順序付きリスト

数値、ドット、半角スペースと入力すると順序付きリストとなる。 

1. 項目1
2. 項目2
3. 項目3
4. 項目4
  1. 項目1
  2. 項目2
  3. 項目3
  4. 項目4

実際は順序は自動採番であるため下記のように振舞う。 

8. 項目1
7. 項目2
6. 項目3
5. 項目4
  1. 項目1
  2. 項目2
  3. 項目3
  4. 項目4

定義リスト

:で始まる定義行と直後のパラグラフで構成される。 HTMLのdl/dt/ddタグ。 

: HTML
Hyper Text Markup Languageの略
: mdoc
軽量マークアップ言語
HTML
Hyper Text Markup Languageの略
mdoc
軽量マークアップ言語

コード(ブロック)

半角スペース4つ、またはタブでインデントする。 

	*****""""````
	+ AAAAAAAAAAAAA
	+ BBBBBBBBBBBBB

mdocのキーワードは全て文字として認識し、 スペース、タブも保存されたまま出力されます。

目次(一部)

[:contents]のみで構成される行を追加すると目次を自動生成します。 [:contents]は記述された時点の見出しレベルを調べ、 同等のレベルの見出しへ到達するまで、下位についてインデックスを生成します。

この文書での目次は見出しの説明の都合上、手作業で作製していますが、 ほとんどの場合は自動作成で事足りるでしょう。 

# 見出し1
[:contents]
## 見出し2
### 見出し3
# 見出し1\`
## 見出し2\`
### 見出し3\`

見出し1
  • 見出し2
    • 見出し3

見出し2

見出し3

見出し1`

見出し2`

見出し3`
